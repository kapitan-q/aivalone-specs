# UserRegistered Event Specification

## Назначение

Событие **UserRegistered** сигнализирует о том, что в системе был успешно создан новый пользователь.
Используется для оповещения других контекстов или подсистем о регистрации нового пользователя.

## Источник события

Вызывается после успешного создания пользователя и сохранения в системе. 

## Состав события

Событие содержит минимальный набор информации, необходимый для идентификации пользователя и его первичной конфигурации:

* `userId` — уникальный идентификатор пользователя (`UserId`)
* `messengers` — массив объектов мессенджеров пользователя, каждый с полями:
  * `messenger` — код мессенджера (`Messenger`)
  * `messengerId` — идентификатор пользователя в мессенджере
* `tariffCodes` — массив кодов тарифов пользователя (по умолчанию `['FREE']`)
* `createdAt` — дата и время создания пользователя

## Применение

* Подписчики события могут:

  * Инициализировать связанные сущности пользователя в других контекстах.
  * Отправлять приветственные уведомления.
  * Логировать факт регистрации.

## Примечания

* Событие является частью Event-Driven архитектуры и предназначено для асинхронного оповещения.
* Содержит только идентифицирующую и агрегированную информацию, не содержит внутренних деталей модели `User`.
* Публикуется только после успешного сохранения пользователя в `UserRepository`.
